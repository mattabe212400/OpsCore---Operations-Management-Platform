# OpsCore

**Internal Operations Management System — Demo Version**

A role-aware web application for managing the day-to-day operations of a chapter-based organization, built with vanilla HTML, CSS, and JavaScript using Firebase for authentication and real-time data sync.

---

## Overview

OpsCore started as a solution to a real problem: trying to manage a large student organization across spreadsheets, group chats, paper sign-in sheets, and disconnected tools.

The goal was to build one place where officers could manage attendance, tasks, finances, recruitment, meetings, and operational tracking — without needing to switch between multiple platforms.

This repository is a public demo version of the original system. It keeps the same overall structure and functionality, but uses sample data and removes all private organizational information.

The project was intentionally built without frontend frameworks to better understand how larger web applications are structured using core web technologies.

## Why I Built This

As someone involved in student organization leadership, I saw how difficult it was to keep operations organized when information was spread across multiple platforms and spreadsheets.

I wanted to build a system that centralized important operational workflows and made it easier for leadership teams to manage information, accountability, and communication in one place.

This project also helped me apply concepts from my MIS coursework to a real operational problem while improving my experience with:

- Frontend development
- Firebase integration
- Real-time systems
- Role-based permissions
- UI/UX design
- Operational workflow planning
